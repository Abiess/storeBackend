# Copilot Dev Notes – markt.ma

## Projekt-Setup & Start-Befehle

### Backend starten
```bash
cd storeBackend
mvn clean compile
```
> Hinweis: Kompiliert das Projekt. Zum Ausführen wird ein separater DB-Start benötigt.

### Frontend starten
```bash
cd storeFrontend
npm start
# → http://localhost:4200
```

### Datenbank
- Lokale H2-Datei: `./data/storedb` (H2 embedded, PostgreSQL-kompatibel)
- H2-Console: `http://localhost:8080/h2-console`
- `spring.jpa.hibernate.ddl-auto=update` → Flyway deaktiviert, neue Felder einfach in Entity ergänzen

### API-Basis-URLs
- Lokal: `http://localhost:8080`
- Produktion: `https://api.markt.ma`
- Auth-Header: `Authorization: Bearer <JWT>`

### WhatsApp-Config (DEV)
- `whatsapp.enabled=false` → Code erscheint im Backend-Log (kein echtes WhatsApp)
- Suche im Log nach: `[DEV] Verification code for +212...`

### Neue Phone-Auth Endpoints (implementiert 2026-06-07)
- `POST /api/auth/phone/request-code` – Code per WhatsApp/Telegram senden (öffentlich, kein JWT)
- `POST /api/auth/phone/verify-and-login` – Code prüfen, JWT zurückgeben (öffentlich)
- Frontend-Route: `/quick-start` (kein authGuard)
